this build uses the corrected DSP core

Unlike the lightweight demo, v3 runs the validated numerical core: a Haar wavelet soft-threshold denoiser for the PRNU residual, FFT-based PCE over the full circular cross-correlation surface, log-polar (Fourier-Mellin) registration for rotation and scale, and Richardson–Lucy deconvolution driven by a PSF estimated from the eye's catchlight. Everything is client-side and runs on a 256×256 power-of-two tile.

Performance is the honest cost. A hand-written radix-2 FFT and log-polar search in JavaScript is far slower than the Python toolkit's compiled libraries. Keep rot+scale registration for short clips, and run deconvolution on demand (the button) rather than per frame. For batch/offline work — full wavelet-MMSE denoising, tested Fourier-Mellin, and fast PCE — use the Python package shipped alongside this file.
